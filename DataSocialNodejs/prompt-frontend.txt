Crea el frontend completo de DataSocial, una red social que conecta usuarios con intereses similares. El proyecto ya esta creado con Expo (expo-router con file-based routing). Usa la base existente y completa todo lo que falta.

EL LOGOTIPO DE LA APLICACIÓN SE LLAMA LOGO Y ESTÁ EN LA CARPETA ASSETS

======================
TECNOLOGIAS OBLIGATORIAS
======================

- React Native con TypeScript
- Expo Router (file-based routing, carpeta app/)
- Axios para peticiones HTTP
- React Native Paper (componentes UI con Material Design)
- NativeWind (Tailwind CSS para React Native)
- Zustand (estado global, sin Redux)
- Splash Screen de Expo (expo-splash-screen)

======================
CONEXION CON EL BACKEND
======================

El backend es una API REST en Node.js/Express corriendo en el puerto 3000.

Configurar la IP base de Axios asi:

import { Platform } from 'react-native';
const IP = Platform.OS === "android" ? "10.0.2.2" : "localhost";
const API_URL = `http://${IP}:3000/api`;

Crear un archivo api/axios.ts con una instancia de Axios configurada con esa baseURL.
NO se usa CORS, no hay tokens ni autenticacion con JWT. El login simplemente guarda el usuario en Zustand.

======================
MODELO DE DATOS
======================

El backend devuelve estos tipos (sin password, nunca llega al front):

interface User {
  id: string;
  nombre: string;
  apellidos: string;
  sexo: 'masculino' | 'femenino' | 'otro' | 'prefiero_no_decir';
  biografia: string;
  gustoPrincipal1: string;
  gustoPrincipal2: string;
  gustoPrincipal3: string;
  email: string;
  fechaCreacion: string;
}

interface Post {
  id: string;
  contenido: string;
  fechaCreacion: string;
  hashtags: string[];
  autor?: string;        // viene en GET /api/posts (nombre + apellidos)
}

interface AfinidadResult {
  id: string;
  nombre: string;
  apellidos: string;
  gustosComunes: string[];
  hashtagsComunes: string[];
  numGustos: number;
  numHashtags: number;
  puntuacionAfinidad: number;
}

======================
ENDPOINTS DE LA API
======================

USUARIOS:
  POST   /api/users/registro   -> Body: { nombre, apellidos, sexo, biografia, gustoPrincipal1, gustoPrincipal2, gustoPrincipal3, email, password }
                                  Respuesta: User (201)
                                  Errores: 400 (campos faltantes, sexo no valido, biografia no valida), 500 (email duplicado)

  POST   /api/users/login      -> Body: { email, password }
                                  Respuesta: User (200)
                                  Errores: 400 (campos faltantes), 401 (credenciales incorrectas)

  GET    /api/users             -> Respuesta: User[] (200)

  GET    /api/users/:id         -> Respuesta: User (200) o 404

  GET    /api/users/:id/afinidad -> Respuesta: AfinidadResult[] (200)
                                    Devuelve ranking de usuarios ordenado por afinidad combinada (gustos + hashtags en comun)

  PUT    /api/users/:id         -> Body: campos parciales (no permite cambiar password)
                                  Respuesta: User (200) o 404

  DELETE /api/users/:id         -> Respuesta: { mensaje: 'Usuario eliminado correctamente' } (200) o 404

POSTS:
  POST   /api/posts             -> Body: { contenido, hashtags: string[], userId }
                                  Respuesta: Post con hashtags (201)
                                  Errores: 400 (campos faltantes, hashtags no es array), 500 (userId no existe)

  GET    /api/posts             -> Respuesta: Post[] con hashtags y autor (200)
                                  Ordenados por fechaCreacion DESC

  GET    /api/posts/usuario/:userId -> Respuesta: Post[] con hashtags (200)

  DELETE /api/posts/:id         -> Respuesta: { mensaje: 'Post eliminado correctamente' } (200) o 404

======================
ZUSTAND STORE
======================

Crear store/useAuthStore.ts con Zustand:

Estado:
  - user: User | null (usuario logueado)
  - isLoggedIn: boolean (derivado de user !== null)

Acciones:
  - login(email, password): llama POST /api/users/login, guarda el user en el store
  - registro(datos): llama POST /api/users/registro, guarda el user en el store (auto-login tras registro)
  - logout(): pone user a null
  - actualizarPerfil(datos): llama PUT /api/users/:id y actualiza el user en el store

No necesita persistencia con AsyncStorage. Al cerrar la app se pierde la sesion.
